Project Hello Quad

3D Graphics · WebGL

Project Overview

Hello Quad is the foundational WebGL project in this graphics series. It establishes the core rendering pipeline: setting up a WebGL context, writing vertex and fragment shaders in GLSL, uploading geometry (a simple quad) to the GPU via VBOs, and issuing draw calls. The goal was to understand the full journey from raw vertex data on the CPU to coloured pixels on the screen.

Tasks Completed

  • Initialised a WebGL 2.0 context and handled canvas resize events.
  • Authored vertex and fragment shaders in GLSL ES 3.00, compiled and linked a shader program.
  • Uploaded quad geometry (positions + UVs) using Vertex Buffer Objects (VBO) and a Vertex Array Object (VAO).
  • Passed transformation matrices (model / view / projection) as uniforms.
  • Implemented a basic render loop with requestAnimationFrame.

Reflection

Getting the first triangle on screen is deceptively tricky — mismatched attribute locations, wrong buffer bindings, and transposed matrices all produce the same black screen. Stepping through the pipeline methodically taught me how each stage connects and made debugging subsequent projects much faster.
